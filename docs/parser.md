# Парсер ЕИС (zakupki.gov.ru)

Файл: `parsers/zakupki.py`

---

## Поиск закупок

**URL поиска:** `https://zakupki.gov.ru/epz/order/extendedsearch/results.html`

### Ключевые параметры запроса

| Параметр | Значение | Описание |
|----------|----------|---------|
| `fz44` | `on` | Включить 44-ФЗ |
| `fz223` | `on` | Включить 223-ФЗ |
| `af` | `on` | Способ: Аукцион в электронной форме |
| `ca` | `on` | Способ: Открытый конкурс |
| `pa` | `on` | Способ: Запрос предложений |
| `searchString` | текст | Ключевые слова + ИНН заказчика + кастомный ОКПД2 |
| `okpd2Ids` | `8873870` | Внутренний числовой ID раздела ОКПД2 |
| `okpd2IdsWithNested` | `on` | Включить вложенные коды |
| `publishDateFrom/To` | `ДД.ММ.ГГГГ` | Дата размещения (date_type=published) |
| `updateDateFrom/To` | `ДД.ММ.ГГГГ` | Дата обновления (date_type=updated) |
| `applSubmissionCloseDateFrom/To` | `ДД.ММ.ГГГГ` | Дата окончания подачи (date_type=end) |
| `priceFromGeneral` | число | НМЦ от (руб) |
| `priceToGeneral` | число | НМЦ до (руб) |
| `pageNumber` | число | Страница (начиная с 1) |
| `recordsPerPage` | `_10` | 10 результатов на страницу |
| `sortBy` | `UPDATE_DATE` | Сортировка по дате обновления |
| `morphology` | `on` | Морфологический поиск |

**Важно:** `customerFullNameOrinn` и `customerTitle` ЕИС игнорирует в расширенном поиске — требует внутренний `customerIdOrg` из JS-модалки. ИНН/наименование заказчика передаётся через `searchString`.

### Внутренние коды ОКПД2 (ЕИС)

| Раздел | Описание | Key |
|--------|----------|-----|
| J | Информация и связь (ИТ) | `8873870` |
| Другие | — | Смотреть в `/epz/nsi/okpd2/open.html` |

## Документы закупки

**Стратегия получения:**
1. Пробуем вкладку документов: `common-info.html` → `documents.html`
2. Если пусто — fallback на саму `common-info.html`

**Фильтрация ссылок:**
- Маркеры файлов: `filestore`, `download`, `/file.`, `.pdf`, `.doc`, `.zip`, `.xls`, `.rar`
- Исключения (`SKIP_URLS`): `zakupki-traffic.xlsx`, `rpt/cat02`, `listModal.html`
- Навигационные метки (`NAV_LABELS`): `информация и документы`, `документы`, `сведения` и др. — заменяются на имя файла из URL

**Приоритет документов для анализа** (сортировка `sort_by_priority`):
1. "описание объекта"
2. "техническое задание" / "тз"
3. "объект закупки"
4. "требования"
5. Остальные

## Парсер запросов цены (pricereq)

Файл: `parsers/priceplan.py`  
**URL:** `https://zakupki.gov.ru/epz/pricereq/search/results.html`

| Параметр | Значение | Описание |
|----------|----------|---------|
| `published` | `on` | Статус: Опубликован |
| `proposed` | `on` | Статус: Предложения поданы |
| `ended` | `on` | Статус: Завершён |
| `publishDateFrom/To` | `ДД.ММ.ГГГГ` | Дата размещения |
| `updateDateFrom/To` | `ДД.ММ.ГГГГ` | Дата обновления |
| `customerTitle` | текст | Наименование заказчика |
| `searchString` | текст | Ключевые слова / объект закупки |

ОКПД2 этим разделом не поддерживается (проверено — возвращает 0 результатов).

## Важные особенности

- **Без прокси**: `proxies={'http': None, 'https': None}` — системный прокси Windows ломает TLS с ЕИС
- **Таймаут**: 45 сек на запрос (`requests.get(..., timeout=45)`)
- **Пауза между страницами**: 1.2 сек (защита от блокировки)
- **Лимит документов**: не более 5 файлов на закупку при batch-скачивании
- **Лимит текста для анализа**: 40 000 символов суммарно по всем файлам
- **Форматы**: `.docx`, `.pdf` (`.doc` читается через python-docx как fallback)
